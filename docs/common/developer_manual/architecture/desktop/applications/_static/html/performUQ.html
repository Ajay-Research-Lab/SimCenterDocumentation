bknd-0036
Dakota-FEM

Uses the uncertainty quantification package Dakota to sample random variables and execute the workflow driver file.

CLI
Inputs 

              
            

            

              

                

                
                  
filenameBIM<string>

                  
File containing a dictionary of building information.

                
                  
filenameEVENT<string>

                  
File containing event intensity measure for each simulation per building.

                
                  
filenameSAM<FileName(r)>

                  
File containing model information for response simulation.

                
                  
filenameEDP<FileName(rw)>

                  
File containing type, location, and direction of recorded EDPs from the response simulation.

                
                  
filenameSIM<FileName(r)>

                  
File containing constants, settings, and methods for the response simulation. This file is deleted after the application is executed.

                
                  
driverFile<FileName(r)>

                  
Batch file containing commands for Dakota to execute, located in the working directory for each simulation.

                
                  
method<string>

                  
Defines the UQ sampling method to use.

                
                  
samples<int>

                  
Specifies the number of realizations to generate. For surrogate models, these samples are used to train the model.

                
                  
seed<int>

                  
Specifies the random seed for sampling.

                
                  
samples2<int>

                  
Only for surrogate models. Specifies the number of realizations to generate from the trained model.

                
                  
seed2<int>

                  
Only for surrogate models. Specifies the random seed to generate samples from the models.

                
                  
ismethod<string>

                  
Specifies the importance sampling method to use

                
                  
dataMethod<string>

                  
...

                
                  
dataMethod2<string>

                  
...

                
                  
type<string>

                  
...

                
                  
concurrency<int>

                  
Specifies the number of processes to use for simulation.

                
                  
keepSamples<bool>

                  
True if the intermediate results shall be kept (not deleted) for each sample.

                
                  
runType<string>

                  
...
